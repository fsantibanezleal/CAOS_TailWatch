# data/ — the data contract + layout

Governed by the **two data contracts** of ADR-0057, instantiated for TailWatch's InSAR-deformation domain.

## Layout

| Path | What | Git |
|---|---|---|
| `raw/scenes/*.h5` | the 20 synthetic Sentinel-1 scenes (the forward sim's output, 168 MB) | **git-ignored** (regenerable from `twlab/science/forward.py`; never committed) |
| `examples/scenes.csv` | a tiny sample of scene descriptors that PASS Contract 1 (clone-verify) | committed |
| `derived/{cnn,ae}.onnx` | the 1-D CNN classifier + the conv-AE anomaly model, exported for live inference | committed (small) |
| `derived/tw-<id>.bin` | the decimated per-case replay cube (8 f32 maps + an i16 cumulative-Up cube) | committed |
| `derived/tw-cases.json` | the rich manifest (grid, classes, forecast, the 5 cases, the held-out benchmark) | committed |
| `derived/forecast-benchmark.json` | LEGACY forecaster self-validation from a pre-0.09 run: degenerate + no in-repo generator; superseded by the `forecast` block in `tw-cases.json`; under re-evaluation (issue #24) | committed |
| `derived/<case>/trace.json` | the compact per-case Contract-2 trace | committed (regenerated by the pipeline) |
| `derived/manifests/` | per-case `<case>.json` (Contract 2) + the flat `index.json` | committed |

The committed `derived/` artifacts ARE the offline lane's real outputs — the DEFAULT pipeline (`python -m
twlab.pipeline all`, numpy-only) rebuilds the per-case traces + manifests from `tw-cases.json`, so a clone replays
without torch or the 168 MB scenes. `pipeline all --retrain` regenerates everything from the forward simulation.

## CONTRACT 1 — ingestion (raw InSAR scene → pipeline) — the *bring-your-own-scene* gate

Defined in `data-pipeline/twlab/io/contract.py`. A scene descriptor is **accepted** iff it satisfies the schema;
ill-formed scenes are **rejected** with a reason; suspicious ones are **flagged** (decorrelated regime; implausible
severity).

| Column | Unit | Rule |
|---|---|---|
| `scene_id` | — | non-empty |
| `W`, `H` | px | [32, 4096] |
| `n_ep` | acquisitions | [3, 500] (SBAS needs enough) |
| `regime` | — | ∈ {stable, linear, accelerating, seasonal, step, decorrelated}; else **reject** |
| `dam_sev` | — | ≥ 0; > 4 → **flag** |
| `coherence_threshold` | — | [0, 1] (optional) |

## CONTRACT 2 — artifact (pipeline → web)

Each run writes a compact per-case trace (`derived/<case>/trace.json`, schema `tailwatch.trace/v1`) + a manifest
(`derived/manifests/<case>.json`, schema `tailwatch.manifest/v2`) recording the category/regime, seed, engine+version,
the shared ONNX + cube, the trace byte size, the lane/gate verdict, the Contract-1 flags, and the case metrics.
`frontend/src/lib/contract.types.ts` mirrors these; `scripts/check_artifacts.py` (CI) enforces byte-consistency.

## Provenance / honesty

The scenes are **synthetic Sentinel-1 simulations** (von-Kármán APS + decorrelation + DEM-error + orbital ramps),
NOT real SAR — TailWatch is didactic + decision-support, not a certified alarm. The held-out split is by scene
(17-20). Real tailings-dam failures (Brumadinho 2019, Cadia 2018) are documented as cautionary/forecastable analogs,
not re-hosted data. No fabricated benchmark numbers.
